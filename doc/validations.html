Validations

var V = FormValidator.Validations;

Assignement

In a field initialization, you may fill parameter called validations with one of the validations.

Representation

Validations may be called with a second argument, repr. It's a value that represents the instance of the validation, and what will be in the valids and invalids fields of the properties parameter of the onInit and onChange callbacks of a FormField.

List of the existing validations

Those validations are available by default.

Unparametrized

EMAIL
The value must be a valid e-mail adress
EMPTY
The value must be empty
NOT_EMPTY
The value must not be empty

Parametrized

All the parameters must be assigned in objects with their given name.
All the parameters are and must be optional.

VALUE_IN ({ min, max })
The value is between min and max.
DATE_BETWEEN ({ min, max })
The date is between min and max.
CHARACTERS ({ count, min, max })
The values contains exactly count characters or more than min characters and less than max characters. count has priority over the other parameters.
SPECIAL_CHARACTERS ({ count, min, max })
The values contains exactly count special characters or more than min special characters and less than max special characters. count has priority over the other parameters.
LETTERS ({ count, min, max })
The values contains exactly count letters or more than min letters and less than max letters. count has priority over the other parameters.
UPPERCASES ({ count, min, max })
The values contains exactly count uppercase letter or more than min uppercase letter and less than max uppercase letter. count has priority over the other parameters.
LOWERCASES ({ count, min, max })
The values contains exactly count lowercase letter or more than min lowercase letter and less than max lowercase letter. count has priority over the other parameters.
NUMBERS ({ count, min, max })
The values contains exactly count digits or more than min digits and less than max digits. count has priority over the other parameters.

Create new validations

For creating new validations, we use V.new. Then you can use your custom validation like the other ones. 

V.new(validationName, validateFunction, hasParams = true)

E.g.

  // ES5
  // With parameters
  V.new(
    'REGEX',
    function(params) {
      return function(value) {
        return params.regex.test(value);
      }
    }
  );

  // Without parameters
  V.new(
    'TEMPLATE',
    function(value) {
      return /^bob$/.test(value);
    },
    false
  )

  // ES6 / ES2015
  // With parameters
  V.new(
    'REGEX',
    (params) => (value) => params.regex.test(value)
  )

  // Without parameters
  V.new(
    'TEMPLATE',
    (value) => /^bob$/.test(value),
  )
